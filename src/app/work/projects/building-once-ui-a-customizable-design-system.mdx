{/* BAGIAN 1: CHALLENGE */}
<En>
## The Challenge
Printer rental customers often struggle to report technical issues accurately. The manual reporting process via phone calls frequently leads to asset identification errors and delays the Service Level Agreement (SLA) response time for the technical support team.
</En>

<Id>
## Tantangan
Pelanggan sewa printer sering kesulitan melaporkan masalah teknis secara akurat. Proses pelaporan manual melalui panggilan telepon sering menyebabkan kesalahan identifikasi aset dan menunda waktu respons Service Level Agreement (SLA) bagi tim dukungan teknis.
</Id>


{/* BAGIAN 2: SOLUTION */}
<En>
## The Solution
I developed **Newton Ezy 2.0**, a web-based customer service portal designed to bridge the gap between hardware infrastructure and software management. The system delivers a seamless "Scan & Report" experience, eliminating manual data entry errors.
</En>

<Id>
## Solusi
Saya mengembangkan **Newton Ezy 2.0**, portal layanan pelanggan berbasis web yang dirancang untuk menjembatani kesenjangan antara infrastruktur perangkat keras dan manajemen perangkat lunak. Sistem ini memberikan pengalaman "Scan & Report" yang mulus, menghilangkan kesalahan entri data manual.
</Id>


{/* BAGIAN 3: FEATURES */}
<En>
## Key Features
- **Instant Asset Identification:** Customers simply scan the physical barcode/QR code on the printer using their smartphones.
- **Direct Reporting Flow:** Upon scanning, the system automatically redirects the user to a specific reporting form for that unit, with the Asset ID pre-filled.
- **Ticket Tracking:** Customers can monitor the repair status and technician progress in real-time.
</En>

<Id>
## Fitur Utama
- **Identifikasi Aset Instan:** Pelanggan cukup memindai barcode/kode QR fisik pada printer menggunakan smartphone mereka.
- **Alur Pelaporan Langsung:** Setelah memindai, sistem secara otomatis mengarahkan pengguna ke formulir pelaporan khusus untuk unit tersebut, dengan ID Aset yang sudah terisi otomatis.
- **Pelacakan Tiket:** Pelanggan dapat memantau status perbaikan dan kemajuan teknisi secara real-time.
</Id>


{/* BAGIAN 4: TECH (SUDAH DIPERBAIKI) */}
<En>
## Technologies Used
- **Laravel**: Robust PHP framework for the backend logic.
- **MySQL**: Database management for asset inventory and ticketing history.
- **Bootstrap**: Responsive UI framework ensuring compatibility across mobile and desktop devices.
- **QR Code Library**: Integration for generating and scanning unique asset codes.
</En>

<Id>
## Teknologi yang Digunakan
- **Laravel**: Framework PHP yang andal untuk logika backend.
- **MySQL**: Manajemen database untuk inventaris aset dan riwayat tiket.
- **Bootstrap**: Framework UI responsif yang memastikan kompatibilitas lintas perangkat mobile dan desktop.
- **QR Code Library**: Integrasi untuk pembuatan dan pemindaian kode aset unik.
</Id>
